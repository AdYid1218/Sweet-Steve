# sobel_02_hdmi_sobel

## 一、实验目标

1. 理解 RGB888 图像到 8 bit 灰度的转换原理
2. 掌握 Sobel 卷积核的边缘检测方法
3. 理解 `edge_mem` 帧缓存的作用
4. 掌握固定图片 Sobel 边缘检测结果通过 HDMI 输出的完整数据流

## 二、实验系统架构

### 2.1 硬件平台

- **开发板**：Zynq-7020
- **显示设备**：HDMI 显示器（1280×720 分辨率）
- **时钟**：系统时钟 50MHz（sys_clk），经 video_clock IP 生成像素时钟和 5× 串行时钟

### 2.2 顶层模块结构

顶层模块 `top.v` 例化了三个主要模块：

| 模块 | 类型 | 功能 |
|------|------|------|
| `hdmi_sobel_display` | PL 逻辑 | Sobel 图像处理与 HDMI 时序生成 |
| `video_clock` | Clock Wizard IP | 生成 pixel clock 和 5× serial clock |
| `rgb2dvi_0` | IP 核 | RGB 转 TMDS 差分信号（HDMI 物理层） |

### 2.3 数据流

```
image_rom_128x72 (RGB888)
       │
       ▼
rgb_to_gray (灰度转换)
       │
       ▼
sobel_core (Sobel 边缘检测)
       │
       ▼
edge_mem (一帧边缘缓存, 128×72)
       │
       ▼
hdmi_sobel_display (10× 放大显示)
       │
       ▼
rgb2dvi_0 (TMDS 编码)
       │
       ▼
HDMI 显示器 (1280×720)
```

**缩放关系**：输入图片 128×72，HDMI 输出 1280×720，放大倍数为 10×10。

## 三、关键模块设计与分析

### 3.1 RGB 转灰度模块 (`rgb_to_gray.v`)

**接口信号**：

| 信号名 | 方向 | 位宽 | 说明 |
|--------|------|------|------|
| `clk` | input | 1 | 像素时钟 |
| `rst_n` | input | 1 | 异步复位，低有效 |
| `rgb_valid` | input | 1 | 输入数据有效标志 |
| `r/g/b` | input | 8 | RGB 三通道像素值 |
| `x/y` | input | 16 | 像素坐标 |
| `gray_valid` | output | 1 | 灰度输出有效标志 |
| `gray` | output | 8 | 灰度值 |
| `gray_x/gray_y` | output | 16 | 灰度像素坐标（直通延迟） |

**灰度转换公式**：

```
gray_calc = R × 77 + G × 150 + B × 29
gray = gray_calc[15:8]  （等效于 ÷256）
```

该公式采用 NTSC 标准加权平均的定点近似：

- R 权重：77/256 ≈ 0.301
- G 权重：150/256 ≈ 0.586
- B 权重：29/256 ≈ 0.113

实现了 1 个时钟周期的流水延迟。

### 3.2 Sobel 核心模块 (`sobel_core.v`)

**接口信号**：

| 信号名 | 方向 | 位宽 | 说明 |
|--------|------|------|------|
| `clk` | input | 1 | 像素时钟 |
| `rst_n` | input | 1 | 异步复位，低有效 |
| `frame_start` | input | 1 | 帧开始标志，高有效 |
| `gray_valid` | input | 1 | 输入灰度有效 |
| `gray` | input | 8 | 输入灰度值 |
| `gray_x/gray_y` | input | 16 | 输入灰度坐标 |
| `edge_valid` | output | 1 | 边缘输出有效 |
| `edge_data` | output | 8 | 边缘强度值 |
| `edge_x/edge_y` | output | 16 | 边缘像素坐标 |
| `edge_frame_done` | output | 1 | 一帧边缘计算完成 |

**Sobel 算子**：

Gx 水平梯度卷积核：

```
    [-1   0  +1]
Gx = [-2   0  +2]
    [-1   0  +1]
```

Gy 垂直梯度卷积核：

```
    [-1  -2  -1]
Gy = [ 0   0   0]
    [+1  +2  +1]
```

**边缘强度计算**：

```
Gx = -top0 + top2 - 2×mid0 + 2×mid2 - bot0 + bot2
Gy = -top0 - 2×top1 - top2 + bot0 + 2×bot1 + bot2
|Gx| = Gx[11] ? (~Gx + 1) : Gx   （取绝对值）
|Gy| = Gy[11] ? (~Gy + 1) : Gy   （取绝对值）
mag = |Gx| + |Gy|                （梯度幅值近似）
edge_data = (mag > 255) ? 255 : mag[7:0]   （饱和截断）
```

**Line Buffer 设计**：

- 使用 `line0[0:WIDTH-1]` 和 `line1[0:WIDTH-1]` 两个行缓冲区实现 3×3 滑动窗口
- `prev2_pixel` = `line0[x]`（前两行的像素）
- `prev1_pixel` = `line1[x]`（前一行的像素）
- 当前输入 `gray` 作为第三行
- 9 个寄存器 `top0~2, mid0~2, bot0~2` 构成 3×3 窗口

**边界处理**：

- 图像四周边界像素（第一行/最后一行、第一列/最后一列）的边缘值直接置 0
- 计算完成后通过 `flush_active` 状态发出边界清零数据，并拉高 `edge_frame_done`

### 3.3 HDMI Sobel 显示模块 (`hdmi_sobel_display.v`)

**HDMI 时序参数**（1280×720 @ 60Hz）：

| 参数 | 数值 |
|------|------|
| H_ACTIVE（水平有效） | 1280 |
| H_FP（水平前肩） | 110 |
| H_SYNC（水平同步） | 40 |
| H_BP（水平后肩） | 220 |
| V_ACTIVE（垂直有效） | 720 |
| V_FP（垂直前肩） | 5 |
| V_SYNC（垂直同步） | 5 |
| V_BP（垂直后肩） | 20 |
| H_TOTAL | 1650 |
| V_TOTAL | 750 |

**状态机设计**：

| 状态 | 说明 |
|------|------|
| `ST_IDLE` | 空闲，立即跳转到 `ST_RUN` 并发出 `frame_start` |
| `ST_RUN` | 逐像素扫描 ROM（128×72），产生 `src_issue` 使能信号 |
| `ST_WAIT` | 等待 Sobel 流水线排空，检测 `edge_frame_done` |
| `ST_DISPLAY` | 将 `sobel_done` 置 1，禁止 ROM 读取，从 `edge_mem` 读取边缘数据显示到 HDMI |

**坐标映射与放大**：

```verilog
disp_x = active_x / SCALE_X;  // 10× 水平放大
disp_y = active_y / SCALE_Y;  // 10× 垂直放大
disp_addr = disp_y * 128 + disp_x;  // edge_mem 读地址
```

HDMI 的 1280×720 活跃区域被划分为 128×72 个 10×10 的块，每块显示对应 `edge_mem` 中一个像素的边缘值（最邻近插值放大）。

**RGB 输出映射**：

```verilog
rgb_r = (de && sobel_done) ? edge_pixel : 8'h00;
rgb_g = (de && sobel_done) ? edge_pixel : 8'h00;
rgb_b = (de && sobel_done) ? edge_pixel : 8'h00;
```

三个通道同时输出相同的边缘强度值，实现**灰度边缘显示**（白色边缘，黑色背景）。

### 3.4 edge_mem 帧缓存

- **容量**：128 × 72 = 9216 字节，声明为 `(* ram_style = "block" *) reg [7:0] edge_mem [0:9215]`
- **写入**：当 `edge_valid == 1` 时，将 `edge_data` 写入 `edge_mem[edge_wr_addr]`
- **读取**：在显示阶段，每个时钟周期从 `edge_mem[edge_rd_addr]` 读出 `edge_pixel`
- **作用**：解耦 Sobel 处理速率与 HDMI 显示速率，确保一帧完整的边缘结果被 HDMI 稳定读出

### 4.5 关键数据流分析

![关键数据流](图片/关键数据流说明.png)

数据流中的关键时序关系：

1. **ROM 读取阶段**：`ST_RUN` 状态下，`src_issue` 信号使能 ROM 地址发生器，从 `(0,0)` 到 `(127,71)` 逐行扫描
2. **灰度转换**：ROM 输出的 RGB 数据经 `src_valid_d1` → `src_valid_d2` 两级延迟后，进入 `rgb_to_gray` 模块，产生 1 拍流水延迟的灰度输出
3. **Sobel 处理**：灰度数据进入 `sobel_core`，经行缓冲和 3×3 窗口卷积，产生 `edge_valid`/`edge_data`
4. **边缘存储**：`edge_valid` 有效时，边缘数据写入 `edge_mem[edge_wr_addr]`
5. **HDMI 显示**：`sobel_done` 置 1 后，HDMI 活跃区域从 `edge_mem` 读取边缘数据显示

## 五、验收要点说明

### 5.1 `rgb_to_gray` 的输入输出信号

- **输入**：`clk`, `rst_n`, `rgb_valid`, `r[7:0]`, `g[7:0]`, `b[7:0]`, `x[15:0]`, `y[15:0]`
- **输出**：`gray_valid`, `gray[7:0]`, `gray_x[15:0]`, `gray_y[15:0]`
- **功能**：将 RGB888 像素按 NTSC 加权公式转换为 8 bit 灰度值，坐标直通输出

### 5.2 `sobel_core` 的输入输出信号

- **输入**：`clk`, `rst_n`, `frame_start`, `gray_valid`, `gray[7:0]`, `gray_x[15:0]`, `gray_y[15:0]`
- **输出**：`edge_valid`, `edge_data[7:0]`, `edge_x[15:0]`, `edge_y[15:0]`, `edge_frame_done`
- **功能**：对输入灰度图进行 3×3 Sobel 卷积，输出边缘强度值（0-255），边界像素输出 0

### 5.3 `edge_mem` 的写入和读取时机

- **写入时机**：`edge_valid == 1` 时，在时钟上升沿将 `edge_data` 写入 `edge_mem[edge_wr_addr]`
  - 写入地址 `edge_wr_addr = edge_y[6:0] * 128 + edge_x[6:0]`
- **读取时机**：HDMI 活跃区域期间，每个时钟周期读取 `edge_mem[disp_addr]`
  - 读取地址 `disp_addr = (active_y/10) * 128 + (active_x/10)`

### 5.4 Sobel 输出如何映射为 HDMI 的 R/G/B 三个通道

原始设计中，三个通道输出相同的 `edge_pixel` 值：

```verilog
rgb_r = (de && sobel_done) ? edge_pixel : 8'h00;
rgb_g = (de && sobel_done) ? edge_pixel : 8'h00;
rgb_b = (de && sobel_done) ? edge_pixel : 8'h00;
```

因此边缘在显示器上呈现为**白色到黑色的灰度渐变**。通过修改此映射关系，可实现多种显示效果（如红色边缘标记、反色显示等）。

## 六、扩展实验

本实验包含 4 项扩展选题，代码整合在 `hdmi_sobel_display.v` 的扩展版本中（见 `扩展作业代码/02改.txt`）。核心思路是将 HDMI 屏幕（1280×720）垂直分为三个区域，在同屏同时展示不同阈值与不同显示效果的对比。

### 6.1 图像特征分析与优化策略

在实现扩展功能之前，首先对输入的固定图片（`image_rom_128x72`）进行分析。观察 ROM 数据：

```
image_mem[ 2] = 24'h040002;
image_mem[ 3] = 24'h060003;
image_mem[ 4] = 24'h080005;
...
```

可以看出图像包含大量**平滑渐变色**（相邻像素 RGB 值缓慢递进），而非锐利的边界。这意味着 Sobel 边缘检测算子计算出的梯度幅值（`edge_pixel`）非常小，通常只有 **5 到 20** 左右。

因此直接使用较高的固定阈值（如 30、80、150）会导致几乎所有边缘都被当作噪点滤除，屏幕输出全黑或全白。针对此问题，扩展代码采取了两项优化：

**优化 1：亮度放大器**

```verilog
wire [7:0] edge_amp = (edge_pixel > 8'd31) ? 8'hFF : {edge_pixel[4:0], 3'b000};
```

通过左移 3 位（即 ×8）将微弱的边缘值放大到可视范围。`edge_pixel` 为 10 时放大后为 80，`edge_pixel` 为 20 时放大后为 160。超出 31 的值直接饱和到 255（`8'hFF`），防止溢出。

**优化 2：大幅降低对比阈值**

将三个区域的阈值分别设定为 **8、15、25**，适配微弱梯度图像的特性。

### 6.2 扩展一：固定阈值二值化边缘显示

**实现原理**：设定固定阈值，当边缘强度大于或等于该阈值时输出纯白（`8'hFF`），否则输出纯黑（`8'h00`）。

**代码实现**（右侧区域，阈值 25）：

```verilog
wire [7:0] r3_RGB = (edge_pixel >= 8'd25) ? 8'hFF : 8'h00;
```

**效果**：有效滤除微小噪点，只保留梯度最强的边缘，边缘呈现为锐利的**纯白轮廓**，背景完全黑色（黑白二值图）。阈值越高，保留的边缘越少，只有最强的轮廓线被显示出来。

**验收标准**：HDMI 显示黑白二值边缘图，报告中给出了所用阈值 25。

### 6.3 扩展二：边缘反色显示

**实现原理**：通过 `8'hFF - edge_amp` 将像素值反转。原本暗色的背景反转为纯白，亮色的边缘反转为暗色调。

**代码实现**（中间区域，阈值 15）：

```verilog
wire [7:0] r2_val = (edge_pixel >= 8'd15) ? edge_amp : 8'h00;
wire [7:0] r2_RGB = 8'hFF - r2_val;
```

**效果**：呈现**白底黑边**的素描效果。背景为纯白（`8'hFF - 0 = 8'hFF`），有边缘的位置根据边缘强度呈现不同深度的灰色/黑色（如边缘值 20 经放大器变为 160，反色后为 95），边缘越强颜色越深。视觉效果类似铅笔素描画。

**验收标准**：原黑底白边变成白底黑边，报告中说明了映射关系 `rgb = 8'hFF - edge_amp`。

### 6.4 扩展三：彩色边缘标记

**实现原理**：仅将放大后的边缘值映射到 RGB 的**红色通道（R）**，绿蓝两通道强制置零。

**代码实现**（左侧区域，阈值 8）：

```verilog
wire [7:0] r1_R = (edge_pixel >= 8'd8) ? edge_amp : 8'h00;
wire [7:0] r1_G = 8'h00;
wire [7:0] r1_B = 8'h00;
```

**效果**：边缘在黑色背景上以**红色**突出显示。因为只用了低阈值 8，该区域保留了最多的微弱边缘细节，能看清渐变区域的细微轮廓变化。红色高亮使得边缘与黑色背景的对比度非常高，易于观察。

**验收标准**：边缘使用红色突出显示，RGB 映射关系为 `{R=edge_amp, G=0, B=0}`。

### 6.5 扩展四：Sobel 阈值参数对比（同屏三分屏）

**实现原理**：将 1280 像素宽的屏幕垂直分为三段，各段使用不同阈值和显示方式，实现**同一画面内**对比不同阈值对边缘提取效果的影响。

**分屏布局**：

```
┌──────────────┬──────────────┬──────────────┐
│   左侧区域    │   中间区域    │   右侧区域    │
│  x < 426     │ 426≤x<853   │   x ≥ 853    │
│  阈值: 8     │  阈值: 15    │  阈值: 25     │
│  红色边缘标记 │  反色显示    │  二值化显示   │
│  (细节最多)   │  (素描效果)   │  (最强轮廓)   │
└──────────────┴──────────────┴──────────────┘
```

**代码实现**（分屏选择逻辑）：

```verilog
reg [7:0] out_R, out_G, out_B;
always @(*) begin
    if (active_x < 12'd426) begin
        // 左侧：低阈值 8 + 红色边缘标记
        out_R = r1_R;  out_G = r1_G;  out_B = r1_B;
    end else if (active_x < 12'd853) begin
        // 中间：中阈值 15 + 反色显示
        out_R = r2_RGB; out_G = r2_RGB; out_B = r2_RGB;
    end else begin
        // 右侧：高阈值 25 + 二值化
        out_R = r3_RGB; out_G = r3_RGB; out_B = r3_RGB;
    end
end
```

**阈值选择依据**：

| 区域 | 阈值 | 选值原因 |
|------|------|----------|
| 左侧 | 8 | 最低阈值，保留最多弱边缘细节，适合与红色高亮配合观察微弱渐变边界 |
| 中间 | 15 | 中等阈值，过滤部分弱噪点，配合反色显示呈现中等密度的素描效果 |
| 右侧 | 25 | 最高阈值，仅保留梯度最强的边缘，输出纯白二值图，轮廓最为清晰锐利 |

**验收标准**：在同一 HDMI 画面中对比了 3 个不同阈值（8、15、25）下的显示效果，直观展示了阈值参数对边缘检测结果的显著影响。

### 6.6 扩展代码完整数据流

```
edge_pixel (来自 edge_mem)
       │
       ├── edge_amp = edge_pixel × 8 (亮度放大)
       │
       ├── 左侧区域 (x<426): 阈值 8 判断 → 红色通道输出
       │         r1_R = (edge_pixel >= 8) ? edge_amp : 0
       │         r1_G = 0, r1_B = 0
       │
       ├── 中间区域 (426≤x<853): 阈值 15 判断 → 反色输出
       │         r2_val = (edge_pixel >= 15) ? edge_amp : 0
       │         r2_RGB = 255 - r2_val
       │
       └── 右侧区域 (x≥853): 阈值 25 判断 → 二值化输出
                r3_RGB = (edge_pixel >= 25) ? 255 : 0
```

### 6.7 扩展实验现象分析

![扩展代码显示](图片/扩展代码HDMISOBEL显示照片.jpg)

从 HDMI 显示照片可以观察到：

1. **左侧区域**（红色边缘）：细节最丰富，模糊渐变区域的微弱轮廓全部可见，因为阈值 8 只过滤了极弱的噪声
2. **中间区域**（反色素描）：白底上出现黑灰色线条，视觉效果类似手绘素描，中等阈值 15 过滤了最弱的噪声
3. **右侧区域**（二值化）：只有最强的轮廓被保留，线条最为简洁锐利，阈值 25 展示了"信噪比最高"的边缘提取结果

三个区域从左到右，呈现出"细节由多到少、轮廓由模糊到锐利"的渐变趋势，直观验证了**阈值参数对边缘检测结果的决定性影响**。

### 6.8 小结

在仅修改 1 个 Verilog 文件（`hdmi_sobel_display.v`）中约 40 行代码的前提下，同时实现了 4 项扩展题目的要求：

- 固定阈值二值化（右侧，阈值 25）
- 边缘反色显示（中间，阈值 15）
- 彩色边缘标记（左侧，红色，阈值 8）
- 三阈值同屏对比（左侧 8、中间 15、右侧 25）

此外，针对输入图像梯度弱的特点，引入了亮度放大器（×8）和适配的低阈值组合，确保了所有区域都能清晰显示边缘效果。

## 七、总结

本实验在 Zynq-7020 平台上实现了一套完整的 **FPGA 图像处理流水线**：

1. 从 Block ROM 读取 128×72 RGB888 固定图片
2. 通过加权平均公式将 RGB 转换为 8 bit 灰度
3. 利用两行缓冲器实现 3×3 Sobel 卷积窗口，计算边缘强度
4. 将一帧边缘结果存入 Block RAM
5. 通过 10× 最邻近插值放大到 1280×720 并通过 HDMI 输出

实验验证了 FPGA 在图像处理领域的**流水线并行**和**低延迟**优势，为后续更复杂的图像处理算法（如 Canny 边缘检测、高斯滤波等）奠定了基础。
