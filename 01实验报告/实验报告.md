# **FPGA 实验一：HDMI 图像显示实验报告**

## 一、 实验名称与目标

**实验名称**：基于 FPGA 的 HDMI 图像显示实验。

**实验验证目标**：本实验基于 **黑金 ZYNQ7020 开发板**，验证从 FPGA 片内 **Block ROM** 到 **HDMI 输出**的完整显示链路。实验将一张 `128 × 72` 的 RGB 测试图片存放在 Verilog ROM 中，通过 `hdmi_image_display` 模块生成 **720p（1280 × 720）** 的显示时序，最终经 `rgb2dvi_0` TMDS 编码器驱动 HDMI 显示器，实现固定图片的放大显示。

完成本实验后，应能回答以下四个核心问题：

**1. HDMI 720p 显示时序的基本参数是什么？**

720p（1280 × 720 @ 60 Hz）的 VESA 标准时序参数如下：

| 参数 | 水平（像素） | 垂直（行） |
|:---|:---:|:---:|
| **有效区域（Active）** | 1280 | 720 |
| **前廊（Front Porch, FP）** | 110 | 5 |
| **同步脉冲（Sync）** | 40 | 5 |
| **后廊（Back Porch, BP）** | 220 | 20 |
| **总像素数（Total）** | **1650** | **750** |

行计数器 `h_cnt` 自增遍历全部 1650 个水平像素，场计数器 `v_cnt` 遍历全部 750 行。行同步 `hs` 在 `h_cnt` 位于区间 `[110, 150)` 时拉低（低电平有效），场同步同理。有效显示区域则位于前廊与同步脉冲之后的 Active 区间内。

**2. `video_clock` 和 `rgb2dvi_0` IP 的作用是什么？**

- **`video_clock`**：Clocking Wizard IP 核，将开发板输入的 50 MHz 参考时钟倍频至 **74.25 MHz**（720p 标准像素时钟频率），为整个 HDMI 链路提供稳定的像素级同步时钟。
- **`rgb2dvi_0`**：Xilinx HDMI 输出接口 IP 核，将并行的 `{R, G, B}` 8-bit 像素数据与 `hs`、`vs`、`de` 编码为 **4 路 TMDS 差分串行信号**（3 路数据 + 1 路时钟），驱动标准 HDMI 显示器。

**3. `128 × 72` 图片如何映射到 `1280 × 720` 显示区域？**

原始代码通过 SCALE 参数实现硬件缩放。`SCALE_X = H_ACTIVE / IMG_WIDTH = 1280 / 128 = 10`，`SCALE_Y = V_ACTIVE / IMG_HEIGHT = 720 / 72 = 10`。ROM 地址通过 `(active_y / 10) * 128 + (active_x / 10)` 计算，每 10 × 10 个屏幕像素对应 ROM 中的一个源像素，将小图粗暴拉伸至满屏，形成马赛克效果。

**4. Verilog ROM 中 `24'hRRGGBB` 像素数据的含义是什么？**

`24'hRRGGBB` 是 **24-bit 真彩色**的十六进制表示：`RR` 为红分量（8 bit），`GG` 为绿分量（8 bit），`BB` 为蓝分量（8 bit）。例如 `24'hFF0000` 为纯红，`24'h87CEEB` 为天蓝色，`24'hFFFFFF` 为白色，`24'h000000` 为黑色。


## 二、 实验原理与架构设计

### 2.1 数据流分析

```
image_rom_128x72 (128 × 72 RGB 图片 ROM)
        │
        ▼
hdmi_image_display (720p 时序生成 + 坐标映射 + 显示控制)
        │
        ▼
rgb2dvi_0 (TMDS 串行编码器)
        │
        ▼
HDMI 显示器 (1280 × 720 @ 60 Hz)
```

整个数据流以像素时钟 `clk`（74.25 MHz）为驱动节拍，分三步说明：

1. **Block ROM 读取**：`image_rom_128x72` 被综合为 **Block RAM**，容量 `128 × 72 = 9216` 个 24-bit 像素。当 `video_active` 有效时，`hdmi_image_display` 根据当前扫描坐标计算出 ROM 地址 `image_addr` 并发出读取请求。
2. **显示控制**：`hdmi_image_display` 内部维护行列计数器 `h_cnt` / `v_cnt`，按 720p 时序产生 `hs`、`vs`、`de`。在有效区内将 ROM 像素数据与边框判定、背景颜色按**优先级多路选择器**逻辑合并，输出 `{R, G, B}` 并行数据。
3. **TMDS 编码输出**：`rgb2dvi_0` IP 将并行数据编码为 4 对 TMDS 差分信号，通过 HDMI 连接器驱动显示器。

### 2.2 扩展任务核心代码逻辑

本实验在基础代码之上完成了全部四项可选拓展，关键修改如下（为方便对照，以下标注了代码中的对应修改点）：

#### 【修改点 1】取消 SCALE 放大，增加居中坐标和边框参数

```verilog
localparam IMG_WIDTH = 128;
localparam IMG_HEIGHT = 72;
localparam [11:0] IMG_X_START = (H_ACTIVE - IMG_WIDTH) / 2;  // 1280-128 / 2 = 576
localparam [11:0] IMG_Y_START = (V_ACTIVE - IMG_HEIGHT) / 2; // 720-72 / 2 = 324
localparam [11:0] BORDER_WIDTH = 12'd4; // 边框宽度为 4 个像素
```

**居中坐标计算**：屏幕有效区域 `1280 × 720`，图片尺寸 `128 × 72`，将空白平均分给两侧：
- X 轴起始 = `(1280 − 128) / 2 = 576`
- Y 轴起始 = `(720 − 72) / 2 = 324`

#### 【修改点 2-5】区域判定与坐标映射

```verilog
// 【修改点 2】增加寄存器，用于对齐 ROM 读出的一拍延迟
reg image_area_d0;
reg border_area_d0;

// 【修改点 3-4】计算区域：判断当前像素是否在中心图片区或边框区
assign image_area = (active_x >= IMG_X_START) && (active_x < (IMG_X_START + IMG_WIDTH)) &&
                    (active_y >= IMG_Y_START) && (active_y < (IMG_Y_START + IMG_HEIGHT));

assign border_area = (active_x >= (IMG_X_START - BORDER_WIDTH)) && 
                     (active_x < (IMG_X_START + IMG_WIDTH + BORDER_WIDTH)) &&
                     (active_y >= (IMG_Y_START - BORDER_WIDTH)) && 
                     (active_y < (IMG_Y_START + IMG_HEIGHT + BORDER_WIDTH)) &&
                     (!image_area); // 外扩 4 像素并镂空中间图片区域

// 【修改点 5】坐标映射：取消除法，改为减去起点坐标
assign image_x = image_area ? (active_x - IMG_X_START) : 7'd0;
assign image_y = image_area ? (active_y - IMG_Y_START) : 7'd0;
assign image_addr = {image_y, 7'b0} + {7'd0, image_x}; // image_y * 128 + image_x
```

**边框生成逻辑**：`border_area` 在 `image_area` 基础上向外扩展 `BORDER_WIDTH`（4 像素），并用 `!image_area` 条件排除中间图片本身，形成"中心镂空"的相框效果。

#### 【修改点 6】多级优先级的 RGB 输出

```verilog
// 优先级 1: 图片区域 → 输出 ROM 像素
// 优先级 2: 边框区域 → 输出红色 24'hFF0000
// 优先级 3: 其他有效区域 → 输出天蓝色 24'h87CEEB
// 优先级 4: 消隐区 (!de_reg_d0) → 必须输出黑色 24'h000000
assign rgb_r = de_reg_d0 ? (image_area_d0 ? image_pixel[23:16] :
                            (border_area_d0 ? 8'hFF : 8'h87)) : 8'h00;
assign rgb_g = de_reg_d0 ? (image_area_d0 ? image_pixel[15:8]  :
                            (border_area_d0 ? 8'h00 : 8'hCE)) : 8'h00;
assign rgb_b = de_reg_d0 ? (image_area_d0 ? image_pixel[7:0]   :
                            (border_area_d0 ? 8'h00 : 8'hEB)) : 8'h00;
```

四级优先级：消隐 > 图片 > 边框 > 背景。消隐区强制输出黑色，严格遵守 HDMI 时序规范。

#### 【修改点 7-8】时序对齐：打一拍延迟

```verilog
always @(posedge clk or posedge rst) begin
    if (rst) begin
        image_area_d0 <= 1'b0;
        border_area_d0 <= 1'b0;
    end else begin
        // ...
        image_area_d0 <= image_area;   // 将区域判定延迟一拍
        border_area_d0 <= border_area; // 对齐 ROM 读出
    end
end
```

由于 Block RAM 存在**一个时钟周期的读取延迟**（给出地址后下一个时钟沿才出数据），若直接用组合信号 `image_area` 控制 RGB 选择，区域判定比像素数据提前一拍到达，将导致画面临界处出现 1 像素的颜色错位。通过引入打一拍寄存器，区域信号与数据严格对齐，确保边缘锐利。

#### 【修改点 9】纯 Verilog 的 ROM 字符生成

```verilog
initial begin
    for (i = 0; i < 9216; i = i + 1) begin
        x = i % 128;  y = i / 128;
        // 字符 'A'(x:29~48, y:21~50) 笔划：左右竖 + 顶横 + 中横
        if (x >= 29 && x <= 48 && y >= 21 && y <= 50)
            if ( (x <= 33) || (x >= 44) || (y <= 25) || (y >= 35 && y <= 39) )
                is_A = 1;
        // 'M'(x:53~72, y:21~50) 与 'D'(x:77~96, y:21~50) 类似
        if (is_A || is_M || is_D)
            image_mem[i] = 24'hFFFFFF; // 白色字体
        else
            image_mem[i] = 24'h000000; // 黑色背景
    end
end
```

利用 Vivado 综合器对 `for` 循环的自动展开能力，在编译阶段逐像素初始化 Block ROM，无需 `.coe` 文件或外部工具。


## 三、 实验步骤与工程实现

### 3.1 步骤简述

1. **打开工程**：`sobel_01_hdmi_pattern/sobel_01_hdmi_pattern.xpr`，确认顶层模块为 `top`。
2. **检查 Sources**：确认包含 `top.v`（顶层）、`hdmi_image_display.v`（显示核心）、`video_clock` IP、`rgb2dvi_0` IP 和 `hdmi_out_test.xdc`（管脚约束）。
3. **代入扩展代码**：将扩展后的 `hdmi_image_display.v` 覆盖原文件，并替换 `image_rom_128x72` 的初始化数据。
4. **综合、实现、生成 Bitstream**：依次点击 **Run Synthesis** → **Run Implementation** → **Generate Bitstream**。
5. **上板验证**：连接 JTAG 与 HDMI，通过 **Hardware Manager** → **Program Device** 选择 `top.bit` 下载。
6. **记录数据**：保存 HDMI 显示照片、综合/实现完成截图、**资源利用率截图**和**时序结果截图**。

### 4.4 结果合理性论证

**坐标计算与居中**：`IMG_X_START = 576`、`IMG_Y_START = 324`。1280 × 720 屏幕减去 128 × 72 图片后将剩余像素平分，图 2、图 3 中图案精准居中，证明减法映射完全正确。

**边框外扩与镂空**：`border_area` 在 `image_area` 基础上外扩 4 像素并用 `!image_area` 排除中心，图 2、图 3 中红框紧贴图块外围而不覆盖图片，逻辑正确。

**多级优先级与背景填充**：嵌套三目运算实现四级优先级。最高优先给图片（ROM 数据），次高给边框（红色），最低给剩余有效区（天蓝），消隐区强制黑色。图 2、图 3 中各区域稳定无花屏，证明优先级分配既满足视觉需求，又严格遵循 HDMI 消隐时序。

**时序对齐**：`image_area_d0` 和 `border_area_d0` 将区域判断延迟一拍，与 ROM 数据对齐。图 2、图 3 中所有交界处锐利无错位，验证了流水线对齐设计的必要性。

**ROM 初始化**：Vivado 综合器成功展开 `for` 循环，逐像素初始化 RAM。图 3 中 "AMD" 图案清晰可辨，证明纯 Verilog 生成 ROM 数据的方法完全可行。


## 五、 实验心得与总结

1. **HDMI 显示时序**：深入理解了 720p 标准中 `H_ACTIVE`、`H_FP`、`H_SYNC`、`H_BP` 与行列计数器的配合关系，参数偏差将直接导致黑屏或信号无法识别。
2. **坐标映射与居中算法**：从"除法缩放"到"减法居中"的改进，体现了 FPGA 中简单加减法替代昂贵除法器的工程思想。
3. **流水线对准**：通过解决 BRAM 读取延迟导致的时序错位，深刻体会到**"打一拍"寄存器延迟**在高速数字电路中的关键作用——这是 FPGA 时序收敛的核心理念。
4. **综合器特性运用**：掌握了利用 Vivado 对 `for` 循环的自动展开完成 Block ROM 初始化的方法，摆脱外部工具依赖。

本次实验为后续更复杂的图像处理（灰度转换、Sobel 边缘检测、UART 传图显示等）打下了扎实的显示链路基础。
